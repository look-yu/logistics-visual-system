通用业务ER图示例

erDiagram
    /* 核心实体：用户 */
    USER {
        int id PK "用户ID（主键）"
        varchar username "用户名"
        varchar password "密码（加密）"
        varchar email "邮箱"
        datetime create_time "创建时间"
        datetime update_time "更新时间"
    }

    /* 核心实体：订单 */
    ORDER {
        int id PK "订单ID（主键）"
        int user_id FK "关联用户ID（外键）"
        decimal amount "订单金额"
        varchar status "订单状态（待支付/已支付/已取消）"
        datetime create_time "创建时间"
    }

    /* 核心实体：商品 */
    PRODUCT {
        int id PK "商品ID（主键）"
        varchar name "商品名称"
        decimal price "商品单价"
        int stock "库存数量"
        varchar category "商品分类"
    }

    /* 关联实体：订单项（订单和商品的多对多关联） */
    ORDER_ITEM {
        int id PK "订单项ID（主键）"
        int order_id FK "关联订单ID（外键）"
        int product_id FK "关联商品ID（外键）"
        int quantity "购买数量"
        decimal total_price "该商品小计金额"
    }

    /* 实体关系定义 */
    USER ||--o{ ORDER : "创建"
    ORDER ||--o{ ORDER_ITEM : "包含"
    PRODUCT ||--o{ ORDER_ITEM : "被包含在"

ER图说明：

  • PK = 主键（Primary Key），FK = 外键（Foreign Key）
  • 用户(USER)和订单(ORDER)是一对多关系：一个用户可以创建多个订单，一个订单仅属于一个用户
  • 订单(ORDER)和订单项(ORDER_ITEM)是一对多关系：一个订单包含多个订单项，一个订单项仅属于一个订单
  • 商品(PRODUCT)和订单项(ORDER_ITEM)是一对多关系：一个商品可以出现在多个订单项中，一个订单项仅对应一个商品
